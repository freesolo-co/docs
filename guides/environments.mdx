An **environment** is the task: the data your model sees and the reward that
scores its outputs. Flash loads environments through the Freesolo environment SDK.
You author one locally, publish it to Freesolo's managed Environments Hub, and
reference it in config by its Freesolo environment id.

```toml
[environment]
id = "your-name/your-env"
```

## Scaffold one

```bash
flash env setup
```

This creates `environment.py` with a working starter. `load_environment()`
returns a Freesolo `EnvironmentSingleTurn` with a sample dataset and reward.

```python environment.py
from freesolo.datasets.types import TaskExample
from freesolo.environments import EnvironmentSingleTurn, RewardResult

DATASET = [
    {"input": "What is 2 + 2?", "output": "4"},
    {"input": "What is 3 + 5?", "output": "8"},
]


class CustomEnv(EnvironmentSingleTurn):
    dataset = DATASET

    def build_prompt_messages(self, example: TaskExample, prompt_text: str):
        return [{"role": "user", "content": example.task}]

    def score_response(self, example: TaskExample, response_text: str) -> RewardResult:
        expected = str(example.expected_output or "").strip()
        score = 1.0 if expected and expected in response_text else 0.0
        return RewardResult(score=score, threshold=1.0)


def load_environment(**kwargs) -> CustomEnv:
    return CustomEnv()
```

Replace the dataset and reward with your task:

- **Dataset**: the prompts (and any gold answers) your model trains and is
  evaluated on.
- **Reward**: `score_response` returns a `RewardResult`; that score is the signal
  GRPO optimizes and SFT can also use for target scoring behavior.

For multi-turn tasks, use `EnvironmentMultiTurn` and implement turn-step hooks.

## Structure the package

A published environment is a small folder. Flash packages that folder, imports
its `environment.py` entrypoint on the worker, and calls
`load_environment(**params)`. That function must return a Freesolo SDK
environment. The folder can be named anything; `math/` and
`environments/math/` are both fine as long as the folder you pass to
`flash env push` contains `environment.py` at its root.

```text
math/
  environment.py          # defines load_environment()
  helpers.py              # optional Python helpers
  datasets/
    train.jsonl           # input/output records
    eval.jsonl
  db/
    state.sqlite          # optional local data sidecar
```

The recommended shape is a folder with `environment.py` at the top. Publish the
folder:

```bash
flash env push --name math math
```

Folder upload is important because it carries everything the environment needs:
helper modules, `datasets/`, `data/`, `db/`, `databases/`, `assets/`, and common
data files. Single `.py` file upload is still supported for tiny smoke tests, but
real environments should be pushed as folders.

Keep all imports either from installed packages or from sibling files in the
folder you publish.

## Use the SDK

Your environment code imports from the `freesolo` package. For local authoring
or `--dry-run` checks that import the environment, install it in the same Python
environment you use for Flash:

```bash
uv pip install freesolo
```

For a single-turn task, subclass `EnvironmentSingleTurn`:

```python environment.py
from pathlib import Path

from freesolo.datasets.records import load_task_examples
from freesolo.datasets.types import TaskExample
from freesolo.environments import EnvironmentSingleTurn, RewardResult

ROOT = Path(__file__).parent


class MathEnv(EnvironmentSingleTurn):
    def __init__(self, *, split: str = "train") -> None:
        self.dataset = load_task_examples(ROOT / "datasets" / f"{split}.jsonl")

    def build_prompt_messages(self, example: TaskExample, prompt_text: str):
        return [{"role": "user", "content": example.task}]

    def score_response(self, example: TaskExample, response_text: str) -> RewardResult:
        expected = str(example.expected_output or "").strip()
        score = 1.0 if expected and expected in response_text else 0.0
        return RewardResult(score=score, threshold=1.0)


def load_environment(split: str = "train", **kwargs) -> MathEnv:
    return MathEnv(split=split)
```

The dataset file uses the user-facing `input`/`output` row shape:

```jsonl datasets/train.jsonl
{"input":"What is 2 + 2?","output":"4"}
{"input":"What is 3 + 5?","output":"8"}
```

At runtime, read the same values from `example.record["input"]` and
`example.record["output"]` if you need the raw row. For the common case,
`load_task_examples(...)` maps `input` to `example.task` and `output` to
`example.expected_output`.

## Publish it

Training runs on managed GPUs, so the environment has to be reachable by id.
Publish your local environment folder to the managed Environments Hub:

```bash
flash env push --name math math
```

It prints the published id
(`your-name/math`). Put that in your config's `[environment] id`.

<Note>
  Publishing is fully managed: `flash env push --name ...` uploads your env to
  Freesolo's hosted Environments Hub (published **private**) under one managed
  account, using only your Freesolo API key. Publish the whole folder so local
  data and helper files are included.
</Note>

## Use an existing environment

If you already have a published env id (yours or one shared with you), you don't
need to author anything. Just reference it:

```toml
[environment]
id = "your-name/your-env"
```

To pull it into your local project for authoring and `--dry-run`:

```bash
flash env install your-name/your-env
```

## List what you have

```bash
flash env list
```

Shows installed Hub environments and local environments you can still publish.

## Pass parameters

If your `load_environment(**kwargs)` accepts arguments, set them under
`[environment.params]`:

```toml
[environment]
id = "your-name/your-env"

[environment.params]
difficulty = "hard"
num_examples = 500
```

<Tip>
  Single-turn environments (one prompt to one response, graded by a reward) are
  fully supported for SFT, GRPO, and eval. Start there, then grow the reward.
  See [Datasets](/guides/datasets) for packaged dataset files and sidecars.
</Tip>
