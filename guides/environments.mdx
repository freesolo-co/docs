title: "Environments"
description: "Define the task your model trains on and is graded by."

An **environment** is the task: the data your model sees and the reward that
scores its outputs. Flash loads environments through the Freesolo environment SDK.
You author one locally, publish it to Freesolo's managed Environments Hub, and
reference it in config by its GitHub environment ref.

```toml
[environment]
id = "github:freesolo-co/environment-hub@main:you/my-task/<publish-id>/environment.py"
```

## Scaffold one

```bash
flash env init my-task
```

This creates `environments/my-task/my_task.py` with a working starter:
`load_environment()` returns a Freesolo `EnvironmentSingleTurn` with a sample dataset
and reward.

```python environments/my-task/my_task.py
from freesolo.datasets.types import TaskExample
from freesolo.environments import EnvironmentSingleTurn, RewardResult

DATASET = [
    {"task": "What is 2 + 2?", "expected_output": "4"},
    {"task": "What is 3 + 5?", "expected_output": "8"},
]


class CustomEnv(EnvironmentSingleTurn):
    dataset = DATASET

    def build_prompt_messages(self, example: TaskExample, prompt_text: str):
        return [{"role": "user", "content": example.task}]

    def score_response(self, example: TaskExample, response_text: str) -> RewardResult:
        expected = str(example.expected_output or "").strip()
        score = 1.0 if expected and expected in response_text else 0.0
        return RewardResult(score=score, threshold=1.0)


def load_environment(**kwargs) -> CustomEnv:
    return CustomEnv()
```

Replace the dataset and reward with your task:

- **Dataset**: the prompts (and any gold answers) your model trains and is
  evaluated on.
- **Reward**: `score_response` returns a `RewardResult`; that score is the signal
  GRPO optimizes and SFT can also use for target scoring behavior.

For multi-turn tasks, use `EnvironmentMultiTurn` and implement turn-step hooks.

## Publish it

Training runs on managed GPUs, so the environment has to be reachable by id.
Publish your local environment to the managed Environments Hub:

```bash
flash env push environments/my-task/my_task.py
```

It prints the published id
(`github:freesolo-co/environment-hub@main:<namespace>/<project>/<publish-id>/environment.py`).
Put that in your config's `[environment] id`.

<Note>
  Publishing is fully managed: `flash env push` uploads your env to Freesolo's
  hosted Environments Hub (published **private**) under one managed account, using
  only your Freesolo API key. By default this writes to `freesolo-co/environment-hub`;
  override with `FLASH_ENV_GITHUB_REPO` if you use a private repo.
</Note>

## Use an existing environment

If you already have a published env ref (yours or one shared with you), you don't
need to author anything. Just reference it:

```toml
[environment]
id = "github:freesolo-co/environment-hub@main:you/my-task/<publish-id>/environment.py"
```

To pull it into your local project for authoring and `--dry-run`:

```bash
flash env install github:freesolo-co/environment-hub@main:you/my-task/<publish-id>/environment.py
```

## List what you have

```bash
flash env list
```

Shows installed Hub environments and local environments you can still publish.

## Pass parameters

If your `load_environment(**kwargs)` accepts arguments, set them under
`[environment.params]`:

```toml
[environment]
id = "github:freesolo-co/environment-hub@main:you/my-task/<publish-id>/environment.py"

[environment.params]
difficulty = "hard"
num_examples = 500
```

<Tip>
  Single-turn environments (one prompt to one response, graded by a reward) are
  fully supported for SFT, GRPO, and eval. Start there, then grow the reward.
  See [Datasets](/guides/datasets) for packaged dataset files and sidecars.
</Tip>
