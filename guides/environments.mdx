An **environment** is the task: the data your model sees and the rubric that
scores its outputs. A Flash environment is a small Python project — an
`environment.py` plus a `datasets/` folder — that you author locally, publish to
Freesolo's Environments Hub, and reference from your config by its id.

```toml
[environment]
id = "your-name/your-env"
```

## Scaffold one

```bash
flash env setup
```

This writes a ready-to-edit starter project into the current directory:

```
environment.py            # the task + rubric
datasets/
  train.jsonl             # one {"input","output"} record per line
configs/
  rl.toml                 # a GRPO config
  sft.toml                # an SFT config
TRAINING.md               # a playbook for iterating on the run
```

`environment.py` subclasses a Freesolo `EnvironmentSingleTurn`: it builds the
prompt for each example and scores the model's response.

```python environment.py
import json
from pathlib import Path

from freesolo.datasets.types import TaskExample
from freesolo.environments import EnvironmentSingleTurn, RewardResult

DEFAULT_DATASET_PATH = Path(__file__).parent / "datasets" / "train.jsonl"


def load_jsonl(path):
    rows = []
    with Path(path).open() as f:
        for line in f:
            line = line.strip()
            if line:
                rows.append(json.loads(line))
    return rows


class StarterEnv(EnvironmentSingleTurn):
    dataset = load_jsonl(DEFAULT_DATASET_PATH)

    def build_prompt_messages(self, example: TaskExample, prompt_text: str):
        return [{"role": "user", "content": example.input}]

    def score_response(self, example: TaskExample, response_text: str) -> RewardResult:
        expected = str(example.output or "").strip()
        score = 1.0 if expected and expected in response_text else 0.0
        return RewardResult(score=score, threshold=1.0)


def load_environment(dataset_path: str | None = None, **kwargs) -> StarterEnv:
    env = StarterEnv()
    if dataset_path:
        env.dataset = load_jsonl(dataset_path)
    return env
```

Replace the dataset and the scoring with your task:

- **Dataset** — `datasets/train.jsonl`, one record per line. Each record is a
  `{"input": ..., "output": ...}` pair: `input` is what the model is prompted
  with, `output` is the gold answer (SFT trains on it directly, and your rubric
  can use it for GRPO).
- **`build_prompt_messages`** — turn one `TaskExample` into the chat messages the
  model sees. `example.input` is the record's input; `example.output` is its gold
  answer.
- **`score_response`** — return a `RewardResult(score=..., threshold=...)`. For
  GRPO the `score` is the reward the policy is optimized toward; `threshold` is
  the score at/above which the response counts as a pass.

## Publish it

Training runs on managed GPUs, so the environment has to be reachable by id.
Publish your project folder to Freesolo's Environments Hub:

```bash
flash env push --name my-env .
```

It prints the published id (`owner/name`). Put that in your config's
`[environment] id`. The path defaults to the current directory, so running
`flash env push --name my-env` from inside the project does the same thing.

<Note>
  Publishing is fully managed: `flash env push` packages your project
  (`environment.py` plus its `datasets/` and any data files) and uploads it to
  Freesolo's hosted Environments Hub (published **private**) using only your
  Freesolo API key. You don't need a separate account or any other CLI.
</Note>

## Use an existing environment

If you already have a published env id (yours or one shared with you), just
reference it — there's nothing to author:

```toml
[environment]
id = "owner/name"
```

You can also point `id` at a public GitHub repo holding an environment, either
as `github:owner/repo` or a full `https://github.com/owner/repo` URL.

## List what you have

```bash
flash env list
```

Shows the local environment sources in the current project that you can publish
with `flash env push`.

## Pass parameters

If your `load_environment(**kwargs)` accepts arguments, set them under
`[environment.params]` — they're passed straight through as keyword arguments:

```toml
[environment]
id = "your-name/your-env"

[environment.params]
difficulty = "hard"
num_examples = 500
```

## Secrets and extra dependencies

If your environment reads secrets at runtime (say an API key for a tool it
calls), declare the env-var **names** under `[environment] secrets`. Their values
are read from your shell or a local `.env` at submit time and forwarded to the
run securely — they're never written into the stored spec:

```toml
[environment]
id = "your-name/your-env"
secrets = ["SERPAPI_API_KEY"]
```

Flash installs `freesolo` itself on the worker, but it does **not** infer your
environment's other dependencies. List every extra Python package your
`environment.py` imports under `[environment] pip` — anything you omit will be
missing at runtime:

```toml
[environment]
id = "your-name/your-env"
pip = ["beautifulsoup4", "httpx"]
```

<Tip>
  Single-turn environments (one prompt → one response, graded by a rubric) are
  the place to start; they're fully supported for SFT, GRPO, and eval. Grow the
  rubric from there.
</Tip>
