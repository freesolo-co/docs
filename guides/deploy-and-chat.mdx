---
title: "Deploy & chat"
description: "Serve a trained adapter and talk to it over an OpenAI-compatible API."
---

Once a run reaches `done`, deploy its adapter and chat with it, from the CLI or
any OpenAI-compatible client.

`flash deploy` registers your adapter with Freesolo's **managed multi-LoRA
serving service**. There is no serving mode to choose and no per-day idle charge:
you register the adapter, send requests to it, and pay only for prompt and
completion tokens (see [Billing](#billing)).

## Deploy

```bash
flash deploy <run-id>
```

Preview what a deploy would do without creating it:

```bash
flash deploy <run-id> --dry-run
```

## Billing

Serving is **billed per token**. Each request costs its prompt + completion
tokens at the base model's serverless list price plus a 20% markup, debited from
your org's prepaid balance. See the actual per-model prices in
[Supported models](/reference/models#serving-prices). There is no separate
per-day idle or warm-GPU charge.

## Chat from the CLI

```bash
flash chat <run-id> -m "Summarize the plot of Hamlet in two sentences."
```

Options:

| Flag | Default | Effect |
| --- | --- | --- |
| `-m`, `--message` | (required) | The user message to send |
| `--max-tokens` | 512 | Maximum tokens to generate |
| `--temperature` | 0.0 | Sampling temperature |

```bash
flash chat <run-id> -m "Write a haiku about GPUs" --temperature 0.8 --max-tokens 128
```

## Manage deployments

```bash
flash deployments        # list active deployments: GPU and endpoint
flash undeploy <run-id>  # deregister the adapter from serving
```

<Note>
  `flash undeploy` deregisters the adapter so it no longer accepts requests.
  You're only billed per token for requests you actually send.
</Note>

## Use it from your own code

Deployments are OpenAI-compatible, so you can point any OpenAI SDK at the
endpoint and use your Freesolo API key:

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://<your-endpoint>/v1",   # from `flash deployments`
    api_key="<your-freesolo-key>",
)

resp = client.chat.completions.create(
    model="<run-id>",
    messages=[{"role": "user", "content": "Hello!"}],
)
print(resp.choices[0].message.content)
```
