title: "Deploy & chat"

---

Once a run reaches `done`, deploy its adapter and chat with it, from the CLI or
any OpenAI-compatible client.

`flash deploy` registers your adapter with Freesolo's **managed multi-LoRA
serving service**. There is no serving mode to choose: you register the adapter,
send requests to it, and pay per token (see [Billing](#billing)).

## Deploy

```bash
flash deploy <run-id>
```

Preview what a deploy would do without creating it:

```bash
flash deploy <run-id> --dry-run
```

## Deploy a specific checkpoint

Every training step that saved an adapter is independently deployable — not just
the run's final one. List a run's deployable checkpoints:

```bash
flash checkpoints <run-id>
```

Then serve a specific step instead of the final adapter:

```bash
flash deploy <run-id> --step <N>
```

This also rescues a run that stopped mid-RL. If you cancelled a GRPO run (or it
failed) after some good steps, its intermediate checkpoints are still deployable
by step. Deploying a `cancelled` or `failed` run's checkpoint attaches the
serving deployment **without** changing the run's terminal training state, and a
later `flash undeploy` leaves that state untouched.

## Billing

Serving is **billed per token**. Each request costs its prompt + completion
tokens at the base model's serverless list price plus a 20% markup, debited from
your org's prepaid balance. See the actual per-model prices in
[Supported models](/reference/models#serving-prices). There is no separate
per-day idle or warm-GPU charge.

**Cached prompt tokens are billed at a 75% discount.** Prefix caching is always
on: when a request's prompt shares a leading prefix with a recent one — a shared
system prompt, or the growing context of a multi-turn chat — the serving engine
reuses that prefix from cache instead of recomputing it, so those tokens bill at
**25% of the input rate**. This is automatic; there is nothing to deploy or
configure. Only the input prefix can be cached, so completion tokens always bill
at the full rate. Each response reports how many prompt tokens were served from
cache in `usage.prompt_tokens_details.cached_tokens`.

## Chat from the CLI

```bash
flash chat <run-id> -m "Summarize the plot of Hamlet in two sentences."
```

Options:

| Flag | Default | Effect |
| --- | --- | --- |
| `-m`, `--message` | (required) | The user message to send |
| `--max-tokens` | 512 | Maximum tokens to generate |
| `--temperature` | 0.0 | Sampling temperature |

```bash
flash chat <run-id> -m "Write a haiku about GPUs" --temperature 0.8 --max-tokens 128
```

## Manage deployments

```bash
flash deployments        # list active deployments: GPU and endpoint
flash undeploy <run-id>  # deregister the adapter from serving
```

## Use it from your own code

Deployments are OpenAI-compatible, so you can point any OpenAI SDK at the
endpoint and use your Freesolo API key:

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://<your-endpoint>/v1",   # from `flash deployments`
    api_key="<your-freesolo-key>",
)

resp = client.chat.completions.create(
    model="<run-id>",
    messages=[{"role": "user", "content": "Hello!"}],
)
print(resp.choices[0].message.content)
```
