---
title: "Flash"
description: "Train, deploy, and chat with custom LoRA models on managed GPUs, from one CLI."
---

Flash is Freesolo's managed post-training service. You write a short config, run
one command, and Flash fine-tunes a model on a managed GPU, then serves the
result behind an OpenAI-compatible endpoint. There's nothing to host: every
command is a thin call to the hosted control plane, authenticated with your
Freesolo API key.

```bash
flash train config.toml      # fine-tune on a managed GPU
flash deploy <run-id>        # serve the trained adapter
flash chat <run-id> -m "hi"  # talk to it
```

## What you can do

<AccordionGroup>
  <Accordion title="Fine-tune a model on your own task" icon="dumbbell">
    Write a TOML config, run one command, and Flash trains a LoRA adapter on top
    of a supported base model. You pick the model and the task; the platform
    picks and runs the GPU. See [Training](/guides/training).

    ```bash
    flash train config.toml
    ```

  </Accordion>
  <Accordion title="Reward the behavior you want with GRPO" icon="trophy">
    Reinforcement learning has the model generate completions, scores them with
    your [environment's](/guides/environments) reward, and updates toward higher
    reward. Use it when you can score an output but can't hand-write the perfect
    one.

    ```toml
    algorithm = "grpo"
    ```

  </Accordion>
  <Accordion title="Teach a format from examples with SFT" icon="copy">
    Supervised fine-tuning learns from the prompt/answer pairs in your dataset.
    Use it when you already have examples of the output you want.

    ```toml
    algorithm = "sft"
    ```

  </Accordion>
  <Accordion title="Serve it behind an OpenAI-compatible API" icon="comments">
    `flash deploy` registers the adapter with managed multi-LoRA serving, then
    `flash chat` or any OpenAI client can call it with your Freesolo key. See
    [Deploy & chat](/guides/deploy-and-chat).

    ```bash
    flash deploy <run-id>
    ```

  </Accordion>
  <Accordion title="See the cost before you spend" icon="dollar-sign">
    `--cost` prints the pre-flight estimate without submitting. You pay for the
    GPU time you train and the tokens you serve; serving has no per-day idle
    charge.

    ```bash
    flash train config.toml --cost
    ```

  </Accordion>
</AccordionGroup>
