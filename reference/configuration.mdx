---
title: "Configuration reference"
description: "Every field in a Flash training config (TOML)."
---

A run is described by a TOML config passed to `flash train`. The only required
fields are `model` and `environment.id`. `algorithm` defaults to `grpo`, and
everything else has a sensible default. Validate any config locally with
`flash train config.toml --dry-run`.

```toml
model = "Qwen/Qwen3.5-4B"
algorithm = "grpo"

[environment]
id = "owner/name"

[train]
steps = 150
lora_rank = 32
```

<Note>
  **The GPU and the run's artifact storage are fully managed** ‚Äî there is no
  `[gpu]` pin to set and no `hf_repo` to supply. The allocator picks the
  cheapest validated GPU class that fits across providers, and each run gets its
  own private Hugging Face repo provisioned server-side. Unknown config sections
  or `[train]` keys are rejected at parse time, so a typo fails fast rather than
  silently training against defaults.
</Note>

## Top level

| Key         | Type   | Default      | Description                                                                        |
| ----------- | ------ | ------------ | ---------------------------------------------------------------------------------- |
| `model`     | string | ‚Äî (required) | Base model to train a LoRA adapter on, e.g. `Qwen/Qwen3.5-4B`. See `flash models`. |
| `algorithm` | string | `grpo`       | `grpo` (RL) or `sft` (supervised).                                                 |
| `thinking`  | bool   | `false`      | Enable reasoning mode (thinking-capable models only).                              |

## `[environment]`

| Key       | Type         | Default        | Description                                                                                                                                                                  |
| --------- | ------------ | -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `id`      | string       | ‚Äî (required)   | Freesolo environment id, `owner/name` (from `flash env push`). Also accepts `github:owner/repo` or a `github.com` URL.                                                       |
| `params`  | table        | `{}`           | Keyword arguments passed to the env's `load_environment(**kwargs)`.                                                                                                          |
| `pip`     | list[string] | `["freesolo"]` | Extra Python packages the env needs on the worker. Flash always installs `freesolo`; list any other package your `environment.py` imports here (they are not auto-detected). |
| `secrets` | list[string] | `[]`           | Env-var names the env reads at runtime; values are read from your shell / local `.env` at submit and forwarded securely, never stored in the spec.                           |

```toml
[environment]
id = "owner/name"
secrets = ["SERPAPI_API_KEY"]

[environment.params]
difficulty = "hard"
```

## `[train]`

| Key                 | Type   | Default        | Description                                                                                                            |
| ------------------- | ------ | -------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `steps`             | int    | ‚Äî              | Number of training steps (GRPO).                                                                                       |
| `epochs`            | int    | ‚Äî              | Number of epochs (SFT; alternative to `steps`).                                                                        |
| `lora_rank`         | int    | `32`           | LoRA rank.                                                                                                             |
| `lora_alpha`        | int    | `64`           | LoRA alpha.                                                                                                            |
| `learning_rate`     | float  | recipe default | Optimizer learning rate.                                                                                               |
| `batch_size`        | int    | recipe default | Training batch size.                                                                                                   |
| `max_length`        | int    | recipe default | Max sequence length.                                                                                                   |
| `save_every`        | int    | recipe default | Checkpoint cadence (steps).                                                                                            |
| `max_steps`         | int    | ‚Äî              | SFT: cap on optimizer steps (`0` = no cap).                                                                            |
| `max_examples`      | int    | ‚Äî              | SFT: truncate the training dataset to N examples (`0` = no cap).                                                       |
| `init_from_adapter` | string | ‚Äî              | Warm-start from an existing adapter. Must be the full adapter ref `owner/repo:sft\|rl/run_id` shown by `flash status`. |

<Note>
  `hf_repo` is **not** a user field. Each run's adapters and checkpoints stream
  into a private Hugging Face repo that the control plane provisions
  automatically ‚Äî any `hf_repo` you set is ignored.
</Note>

### GRPO-specific

| Key                            | Type         | Description                        |
| ------------------------------ | ------------ | ---------------------------------- |
| `group_size`                   | int          | Completions sampled per prompt.    |
| `temperature`                  | float        | Sampling temperature for rollouts. |
| `max_tokens`                   | int          | Max tokens per rollout completion. |
| `kl_penalty_coef`              | float        | KL penalty coefficient.            |
| `advantage_clip`               | float        | Centered-advantage clamp.          |
| `thinking_length_penalty_coef` | float        | Penalty on reasoning length.       |
| `stop_sequences`               | list[string] | Stop sequences for generation.     |

## GPU & infrastructure (managed)

GPU selection is **fully automatic** ‚Äî there is no GPU pin. At submit time the
allocator always picks the cheapest live-validated GPU class that fits your run
(RunPod, with Lambda Labs as a capacity fallback), and a retry walks to the
next-cheapest class if one is unavailable. Disk, the wall-clock cap, and the
auto-retry budget are platform defaults, also chosen for you. A `[gpu]` table in
your config is **not honored**; to see the classes and live pricing the allocator
chooses from, run `flash gpus`.

## `[wandb]`

Optional [Weights & Biases](https://wandb.ai) logging labels. These values are
non-secret; set `WANDB_API_KEY` in your local environment when submitting a run
to enable logging to your own W&B account.

| Key        | Type   | Description       |
| ---------- | ------ | ----------------- |
| `project`  | string | W&B project name. |
| `run_name` | string | W&B run name.     |

## `[worker_env]`

An optional advanced escape hatch: a table of string-valued environment variables
set on the GPU worker for this run. Secret-bearing keys are rejected (these are
serialized into run artifacts) ‚Äî declare runtime secrets under
`[environment] secrets` instead.

```toml
[worker_env]
SOME_TUNING_FLAG = "1"
```

## Overrides & composition

Any value can be set at submit time without editing the file. The override flag
is `--set` (repeatable, dotted keys):

```bash
flash train config.toml --set train.steps=300 --set train.lora_rank=16
flash train base.toml --config overlay.toml      # deep-merge extra TOML
```
