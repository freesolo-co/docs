Flash trains a **LoRA adapter** on top of a curated catalog of base models. You
set the base model with one line in your config, and list the live catalog from
the CLI:

```bash
flash models
```

```toml
model = "Qwen/Qwen3.5-4B"   # required — see `flash models` for the options
```

`flash models` is always the source of truth. The table below mirrors the
current catalog.

## Catalog

Every catalog model supports **both SFT and GRPO**, and all are
reasoning ("thinking") capable.

| Model (`model =`)     | Reasoning |
| --------------------- | --------- |
| `Qwen/Qwen3.5-0.8B`   | Hybrid    |
| `openbmb/MiniCPM5-1B` | Hybrid    |
| `Qwen/Qwen3.5-2B`     | Hybrid    |
| `Qwen/Qwen3.5-4B`     | Hybrid    |
| `Qwen/Qwen3.5-9B`     | Hybrid    |
| `Qwen/Qwen3.6-35B-A3B` | Hybrid   |

## Serving prices

Serving is billed per token after deployment. Each model has separate **input
(prompt)** and **output (completion)** rates: the base model's serverless list
price plus a fixed 20% markup, with output billed at 2x the input rate. The
prices below are **per 1M tokens**; your Freesolo billing dashboard shows the
charges you actually accrue.

| Model | Input / 1M | Output / 1M | Cached input / 1M |
| --- | ---: | ---: | ---: |
| `Qwen/Qwen3.5-0.8B` | $0.06 | $0.12 | $0.015 |
| `openbmb/MiniCPM5-1B` | $0.06 | $0.12 | $0.015 |
| `Qwen/Qwen3.5-2B` | $0.12 | $0.24 | $0.03 |
| `Qwen/Qwen3.5-4B` | $0.24 | $0.48 | $0.06 |
| `Qwen/Qwen3.5-9B` | $0.60 | $1.20 | $0.15 |
| `Qwen/Qwen3.6-35B-A3B` | $2.40 | $4.80 | $0.60 |

The **Cached input** column is the rate for prompt tokens served from the
**prefix cache** — **25% of the input rate** (a 75% discount) — because the
serving engine reuses a cached prefix instead of recomputing it. Caching is
automatic whenever a request's prompt shares a leading prefix with a recent one
(a shared system prompt, or a multi-turn chat); only prompt tokens are
cacheable, so completion tokens always bill at the full output rate. See
[Billing](/guides/deploy-and-chat#billing).

### Notes per model

- **`Qwen/Qwen3.5-0.8B`**: the smallest Qwen3.5. Cheapest for smoke tests and
  fast iteration.
- **`openbmb/MiniCPM5-1B`**: an on-device-class small model (131k context) on a
  standard Llama architecture.
- **`Qwen/Qwen3.5-2B`**: a small, capable step up when 0.8B underfits.
- **`Qwen/Qwen3.5-4B`**: a balanced starting point for most tasks.
- **`Qwen/Qwen3.5-9B`**: the largest dense catalog model. Use it when quality
  matters more than the run cost.
- **`Qwen/Qwen3.6-35B-A3B`**: a Mixture-of-Experts checkpoint (35B total, ~3B
  active per token) and the largest model in the catalog. It is the
  big-checkpoint tier (SFT runs on an H200, GRPO on a B200), so it costs the
  most per run and per served token.

<Note>
  The Qwen3.5 entries are **text-only** fine-tunes: the checkpoints are natively
  multimodal, but Flash trains and serves the language model only.
</Note>

## Reasoning ("thinking") models

Every catalog model is **hybrid-reasoning**: it can run with or without an
explicit reasoning step. Turn reasoning on for a run with one line in your
config:

```toml
thinking = true
```

See the [`thinking` field](/reference/configuration#top-level) in the
configuration reference.
