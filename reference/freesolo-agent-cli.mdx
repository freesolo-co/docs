---

The `freesolo` CLI runs repository-agent workflows.

- Use **`flash`** to train LoRA adapters on managed GPUs, deploy them, and chat
  with them.
- Use **`freesolo`** for repository-agent workflows that draft, optimize, train,
  or poll backend jobs against a source repo.

## Install

```bash
pip install freesolo-agent
```

The package installs a single command:

```bash
freesolo setup
```

Python 3.11 or newer is required.

## Setup flow

`freesolo setup` is interactive. It prompts for:

- A Freesolo API key.
- The operation to run: `draft`, `optimize`, `training`, or `poll`.
- A source repository URL for new draft jobs.
- A prompt or description for the job.
- Optional environment variables, saved as encrypted org secrets for the backend
  job.

For private GitHub repositories, install the Freesolo GitHub App when prompted:

```text
https://github.com/apps/freesolo-agent/installations/new
```

## Operations

| Operation | What it does |
| --- | --- |
| `draft` | Start a new backend job from a repository URL and task description. |
| `optimize` | Continue from a stored org job/project and run the optimization phase. |
| `training` | Continue from a stored org job/project and run the training phase. |
| `poll` | Follow an existing job until it reaches a terminal state. |

The backend job states are `queued`, `running`, `succeeded`, `blocked`,
`user_stopped`, and `failed`.

## Help

Run `freesolo --help` or `freesolo setup --help` for command help. Use `flash`
for managed LoRA training, deployment, and chat.
