The CLI is `flash` (installed from the **`freesolo-flash`** package — see the
[Quickstart](/quickstart#1-install-the-cli)). Every run-lifecycle command is a
thin, authenticated call to the hosted control plane. Run `flash --help` or
`flash <command> --help` for inline help.

## Global flags

| Flag              | Effect                                                                |
| ----------------- | --------------------------------------------------------------------- |
| `-h`, `--help`    | Show help and exit (works on any subcommand too)                      |
| `-V`, `--version` | Print the Flash version and exit                                      |
| `--debug`         | Show full tracebacks on error (otherwise errors print one clean line) |
| `-v`, `--verbose` | Increase log verbosity (`-v` info, `-vv` debug)                       |

## Auth & identity

<ResponseField
  name="flash login"
  type="[--api-key KEY] [--freesolo-url URL] [--api-url URL]"
>
  Log in with your Freesolo API key. The key is verified against Freesolo and
  saved locally. Create a key at
  [freesolo.co/sign-in](https://freesolo.co/sign-in). Defaults: `--api-key` from
  `FREESOLO_API_KEY`; `--freesolo-url` from `FREESOLO_BASE_URL` (else
  `https://api.freesolo.co`); `--api-url` from `FLASH_API_URL`.
</ResponseField>

<ResponseField name="flash whoami">
  Show the identity your stored key resolves to.
</ResponseField>

<ResponseField name="flash version">
  Print the Flash version (same as `-V` / `--version`).
</ResponseField>

## Discovery

<ResponseField name="flash models">
  List the supported base model ids (the per-model algorithms, quantization, and
  reasoning capability live in the catalog but aren't printed by this command).
</ResponseField>

<ResponseField name="flash gpus">
  List managed GPU classes with VRAM and RunPod `$/hr`. GPU selection is
  automatic — there is no GPU pin (see [GPU
  selection](/guides/training#gpu-selection-is-automatic)).
</ResponseField>

## Environments

<ResponseField name="flash env setup">
  Scaffold a starter environment project in the current directory:
  `environment.py`, a `datasets/` folder, `configs/rl.toml`, `configs/sft.toml`,
  and `TRAINING.md`.
</ResponseField>

<ResponseField name="flash env list">
  List the local environment sources in the current project that you can
  publish.
</ResponseField>

<ResponseField name="flash env push --name NAME [PATH]">
  Publish a local environment project to Freesolo's managed Environments Hub
  (private) and print its id (`owner/name`). `PATH` defaults to the current
  directory. Fully managed — no separate account or other CLI needed.
</ResponseField>

## Training

<ResponseField name="flash train CONFIG [flags]">
  Submit a managed training run from a TOML config and follow its logs.
  Submitting charges the pre-flight cost estimate to your org and is gated on
  balance (see [Cost & billing](/guides/training#cost-and-billing)).

| Flag              | Effect                                                                     |
| ----------------- | -------------------------------------------------------------------------- |
| `--dry-run`       | Validate the config locally and print the resolved job spec; don't submit  |
| `--cost`          | Print the pre-flight USD cost estimate for the config and exit (no submit) |
| `--background`    | Submit and return immediately instead of following logs                    |
| `--set KEY=VALUE` | Override a config value (dotted key, repeatable)                           |
| `--config FILE`   | Deep-merge additional TOML for config composition (repeatable)             |

</ResponseField>

## Run management

<ResponseField name="flash runs">
  List your runs with state, algorithm, accrued cost, GPU, and model.
</ResponseField>

<ResponseField name="flash status RUN_ID [--logs] [-f]">
  Print a run's full status. `--logs` also prints the run's logs (the
  orchestrator log plus the train-subprocess stdout/traceback). `-f`/`--follow`
  streams logs until the run ends, then prints the final status.
</ResponseField>

<ResponseField name="flash checkpoints RUN_ID">
  List a run's deployable per-step RL checkpoints (SFT-only runs have none).
  Deploy one with `flash deploy RUN_ID --step <STEP>`.
</ResponseField>

<ResponseField name="flash cancel RUN_ID">
  Cancel a run (best-effort).
</ResponseField>

## Serving

<ResponseField name="flash deploy RUN_ID [flags]">
  Register a trained adapter with the managed multi-LoRA serving service.
  Serving scales to zero when idle and is [billed per
  token](/guides/deploy-and-chat#billing).

| Flag        | Default | Effect                                                                              |
| ----------- | ------- | ----------------------------------------------------------------------------------- |
| `--step N`  | —       | Deploy a specific checkpoint (see `flash checkpoints`) instead of the final adapter |
| `--dry-run` | —       | Preview the deploy without creating it                                              |

</ResponseField>

<ResponseField name="flash chat RUN_ID -m MESSAGE [flags]">
  Send a message to a deployed adapter and print the reply.

| Flag              | Default    | Effect                 |
| ----------------- | ---------- | ---------------------- |
| `-m`, `--message` | (required) | The user message       |
| `--max-tokens`    | `512`      | Max tokens to generate |
| `--temperature`   | `0.0`      | Sampling temperature   |

</ResponseField>

<ResponseField name="flash deployments">
  List active serving deployments with their run id, GPU, and endpoint.
</ResponseField>

<ResponseField name="flash undeploy RUN_ID">
  Deregister a run's adapter from serving so it no longer accepts requests.
</ResponseField>

## Export

<ResponseField name="flash export --adapter-id ID --repository REPO [flags]">
  Export a trained adapter to a Hugging Face repo you own.

| Flag                      | Default    | Effect                                                                  |
| ------------------------- | ---------- | ----------------------------------------------------------------------- |
| `--adapter-id ID`         | (required) | The Freesolo adapter id (the run id) to export                          |
| `--repository owner/name` | (required) | Destination HuggingFace repo (created if it doesn't exist)              |
| `--api-key TOKEN`         | `HF_TOKEN` | HuggingFace token with write access (from your shell or a local `.env`) |
| `--step N`                | —          | Export a specific checkpoint instead of the run's final adapter         |
| `--public`                | private    | Create the destination repo public                                      |

</ResponseField>
